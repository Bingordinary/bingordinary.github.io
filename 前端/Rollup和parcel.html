# Rollup和parcel

# Rollup

Rollup (opens new window)是一个 JavaScript 模块打包器，可以将小块代码编译成大块复杂的代码，例如 library 或应用程序

Rollup与webpack作用类似，但Rollup更为小巧，仅仅是一款ESM打包器，提供一个充分利用ESM各项特性的高效打包器

# Rollup快速上手

create.js

export const createInput = () => {
    let input = document.createElement('input')
    input.placeholder = "请输入内容"
    return input
}
export const createHead = () => {
    let header = document.createElement('div')
    header.innerHTML = 'header'
    return header
}
export const createButton = () => {
    let button = document.createElement('button')
    button.innerHTML = '按钮'
    return button
}

index.js

import {
    createHead
} from './create'
document.body.append(createHead())

生产一个浏览器环境的文件

rollup src/index.js --format iife --file dist/bundle.js

编译后的代码

(function () {
    'use strict';

    const createHead = () => {
        let header = document.createElement('div');
        header.innerHTML = 'header';
        return header
    };

    document.body.append(createHead());

}());

Rollup打包结果相比webpack没有多余的代码，且自动去除了无用的代码

# 使用配置文件

在项目文件下新建rollup.config.js

export default {
    input:'src/index.js',
    output:{
        file:'dist/bundle.js',
        format:'iife'
    }
}
yarn rollup --config

# 使用插件

插件是Rollup唯一的拓展方式

import json from 'rollup-plugin-json'
export default {
    input: 'src/index.js',
    output: {
        file: 'dist/bundle.js',
        format: 'iife'
    },
    plugins: [
        json()
    ]
}

编译后的文件

(function () {
    'use strict';

    const createHead = () => {
        let header = document.createElement('div');
        header.innerHTML = 'header';
        return header
    };

    var name = "rollup";
    var version = "1.0.0";

    document.body.append(createHead());
    console.log(name, version);

}());

# 加载NPM模块

Rollup默认只能按文件路径方式加载本地模块

使用rollup-plugin-node-resolve加载npm模块

import json from 'rollup-plugin-json'
import resolve from 'rollup-plugin-node-resolve'
export default {
    input: 'src/index.js',
    output: {
        file: 'dist/bundle.js',
        format: 'iife'
    },
    plugins: [
        json(),
        resolve()
    ]
}

# Rollup加载CommonJS模块

一些库导出成你可以正常导入的 ES6 模块 - the-answer 就是一个这样的模块。 但是目前， npm 中的大多数包都是以 CommonJS 模块的形式出现的。 在它们更改之前，我们需要将 CommonJS 模块转换为 ES2015 供 Rollup 处理。

如果想要CommonJS,可以使用rollup-plugin-commonjs这个插件，用来将 CommonJS 转换成 ES2015 模块

import json from 'rollup-plugin-json'
import resolve from 'rollup-plugin-node-resolve'
import commonjs from 'rollup-plugin-commonjs'

export default {
    input: 'src/index.js',
    output: {
        file: 'dist/bundle.js',
        format: 'iife'
    },
    plugins: [
        json(),
        resolve(),
        commonjs()
    ]
}

# Rollup代码拆分

使用上面代码我们直接运行会报错UMD and IIFE output formats are not supported for code-splitting builds.

使用Rollup不支持UMD和IIFE,浏览器端我们可以使用AMD，当我们输出多个文件时，必须指定输出目录output.dir,而不是out.file rollup.config.js

import json from 'rollup-plugin-json'
import resolve from 'rollup-plugin-node-resolve'
import commonjs from 'rollup-plugin-commonjs'

export default {
    input: 'src/index.js',
    output: {
        // file: 'dist/bundle.js',
        // format: 'iife'
        format:'amd',
        dir:'dist'
    },
    plugins: [
        json(),
        resolve(),
        commonjs()
    ]
}

# Rollup选用原则

Rollup的优势 输出结果更加扁平 自动移除未引用代码 打包结果依然可读 缺点 加载非ESM第三方模块比较复杂 模块最终都被打包到一个函数中，无法实现HMR 浏览器环境中，代码拆分功能依赖AMD库

如果我们正在开发应用程序，可以选择webpack;如果我们正在开发一个框架或类库

Webpack大而全，Rollup小而美Parcel

开发启动

yarn parcel src/index.html

Parcel可以自动安装依赖

import _ from 'lodash-es'

处理图片、css等文件不需要任何配置

import './style.css'
<img src="./parcel-front.webp">

使用动态导入会自动拆分代码

    console.log(_.camelCase('hello word'))
})

生产打包

yarn parcel build src/index.html

Parcel的特点：完全零配置、自动安装依赖、构建速度比较快
