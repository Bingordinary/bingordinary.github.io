# JavaScript异步编程

# 同步模式

代码中的任务依次执行，后一个任务必须等待上一个任务结束才能执行。如果其中某个任务执行时间过长，后面的任务就会被阻塞

# 异步模式

不会等待这个任务的结束才开始下一个任务，任务开启过后就立即往后执行下一个任务，后续逻辑一般会通过回调函数的方式定义 缺点是代码的执行顺序混乱

# 回调函数

一个函数被作为参数传递给另一个函数（在这里我们把另一个函数叫做“otherFunction”），回调函数在otherFunction中被调用。

# Promise

Promise (opens new window) 是一个对象，它代表了一个异步操作的最终完成或者失败

# Promise基本用法

const promise=new Promise((resolve,reject)=>{
    // resolve(100)
    reject('err msg')
})

promise.then(res=>{
    // console.log(res)//100

},err=>{
    console.log(err)//err msg  后打印
})

console.log('end')//end  先打印

上面代码中即使promise中没有任何异步操作，then中的函数依然会进入回调队列中排队

# 使用案例

使用promise封装简易的ajax

function ajax(url){
    return new Promise((resolve,reject)=>{
        var xhr=new XMLHttpRequest()
        xhr.open('GET',url)
        xhr.responseType='json'
        xhr.onload=function(){
            if(this.status===200){
                resolve(this.response)
            }else{
                reject(new Error(this.statusText))
            }
        }
        xhr.send()
    })
}
ajax('http://localhost/api/list.json').then(res => {
    console.log(res)
}).catch(err => {
    console.log(err)
})

嵌套使用 Promise 是最常见的误区

      ajax(urls.users).then(function (users) {

  })
})

# Promise的链式调用

var promise = ajax('/api/users.json')

var promise2 = promise.then(
  function onFulfilled (value) {
    console.log('onFulfilled', value)
  },
  function onRejected (error) {
    console.log('onRejected', error)
  }
)

console.log(promise2 === promise)//false
ajax('/api/users.json')
  .then(function (value) {
    console.log(1111)
    return ajax('/api/urls.json')
  }) // => Promise
  .then(function (value) {
    console.log(2222)
    console.log(value)
    return ajax('/api/urls.json')
  }) // => Promise
  .then(function (value) {
    console.log(3333)
    return ajax('/api/urls.json')
  }) // => Promise
  .then(function (value) {
    console.log(4444)
    return 'foo'
  }) // => Promise
  .then(function (value) {
    console.log(5555)
    console.log(value)
  })
  • Promise对象的then方法会返回一个全新的Promise对象
  • 后面的then方法就是在为上一个then返回的Promise注册回调
  • 前面then方法中的回调函数的返回值最为后面then方法回调的参数
  • 如果回调中返回的是Promise,后面的then方法会等待它的结束

# Promise异常处理

一个 Promise有以下几种状态:

  • pending: 初始状态，既不是成功，也不是失败状态。
  • fulfilled: 意味着操作成功完成。
  • rejected: 意味着操作失败。 pending 状态的 Promise 对象可能会变为fulfilled 状态并传递一个值给相应的状态处理方法，也可能变为失败状态（rejected）并传递失败信息。当其中任一种情况出现时，Promise 对象的 then 方法绑定的处理方法（handlers ）就会被调用（then方法包含两个参数：onfulfilled 和 onrejected，它们都是 Function 类型。当Promise状态为fulfilled时，调用 then 的 onfulfilled 方法，当Promise状态为rejected时，调用 then 的 onrejected 方法， 所以在异步操作的完成和绑定处理方法之间不存在竞争）

处理异常的两种写法

ajax('http://localhost/api/list.json').then(res => {
    console.log(res)
},err => {
  console.log(err)
})
// 下面这种写法更常用  可以捕获整个链上的异常
ajax('http://localhost/api/list.json').then(res => {
    console.log(res)
}).catch(err => {
    console.log(err)
})

因为 Promise.prototype.then 和 Promise.prototype.catch 方法返回promise 对象， 所以它们可以被链式调用。

# Promise的静态方法

有时需要将现有对象转为 Promise 对象，Promise.resolve()方法就起到这个作用 Promise.resolve()

Promise.resolve('foo')
// 等价于
new Promise(resolve => resolve('foo'))

  • 参数是一个 Promise 实例

  • 如果参数是 Promise 实例，那么Promise.resolve将不做任何修改、原封不动地返回这个实例。

  • 参数是一个thenable对象

  • Promise.resolve方法会将这个对象转为 Promise 对象，然后就立即执行thenable对象的then方法。

let thenable = {
  then: function(resolve, reject) {
    resolve(42);
  }
};
  • 参数不是具有then方法的对象，或根本就不是对象
  • 如果参数是一个原始值，或者是一个不具有then方法的对象，则Promise.resolve方法返回一个新的 Promise 对象，状态为resolved。
const p = Promise.resolve('Hello');
  p.then(function (s){
    console.log(s)
  })
// Hello
  ```

# Generator异步方案

整个 Generator 函数就是一个封装的异步任务，或者说是异步任务的容器。异步操作需要暂停的地方，都用yield语句注明

function* foo() {
    console.log('start')
    let res = yield 'foo'
    console.log(res) //red 外部传入的值
}
const generator = foo()
console.log('before')
const res = generator.next()
console.log(res) //{ value: 'foo', done: false }
generator.next('red')
// 打印结果
// before
// start
// { value: 'foo', done: false }
// red
  • next方法的作用是分阶段执行Generator函数
  • next返回值的 value 属性，是 Generator 函数向外输出数据；next方法还可以接受参数，向 Generator 函数体内输入数据。

Generator 函数内部还可以部署错误处理代码，捕获函数体外抛出的错误。 throw方法也可以让函数内部继续执行，抛出的异常函数内部try...catch捕获到

function* gen(x){
  try {
    var y = yield x + 2
  } catch (e){
    console.log(e)
  }
  return y
}

var g = gen(1)
g.next()
g.throw('出错了')
// 出错了
// 先执行 Generator 函数，获取遍历器对象，然后使用next方法，执行异步任务的第一阶段。由于ajax返回的是一个 Promise 对象，因此要用then方法调用下一个next方法
let g = main()

let result = g.next()
result.value.then(data => {
    let result2 = g.next('result2')
    if (result2.done) {
        return
    }
    result2.value.then(res => {
        let result3 = g.next('result3')
        if (result3.done) {
            return
        }
        console.log(result3)
    })
})

使用递归执行Generator函数

function handleResult(g) {
    let res = g.next()
    if (res.done) {
        return
    }
    res.value.then(data => {
        console.log(data)
        handleResult(g)
    },err => {
        g.throw(err)
    })
}
let g = main()

handleResult(g)

将上面代码改造，模拟实现co (opens new window)

function co(genrator) {
    const g = genrator()
    handleResult()
    function handleResult() {
        let res = g.next()
        if (res.done) {
            return
        }
        res.value.then(data => {
            console.log(data)
            handleResult()
        }, err => {
            g.throw(err)
        })
    }
}
co(main)

# async 函数

async是 Generator 函数的语法糖 async函数返回一个 Promise 对象，可以使用then方法添加回调函数。当函数执行的时候，一旦遇到await就会先返回，等到异步操作完成，再接着执行函数体内后面的语句

async function main() {
    const user = await ajax('http:localhost/api/list.json')
    console.log(user)
    const data = await ajax('http:localhost/api/data.json')
    console.log(data)
}

注意：await只能出现在async函数内部使用
