# TypeScript

# 强类型与弱类型

强类型 语言层面限制函数的实参类型必须与形参类型相同，不允许任意的隐式类型转换 弱类型 不限制实参的类型，允许任意的隐式类型转换

# 静态类型和动态类型

静态类型 声明时类型就是明确的，声明过后，类型不允许修改 动态类型 运行阶段才能明确变量类型，变量的类型随时可以改变。变量没有类型，变量存放的值是有类型的

# JavaScript类型系统特征

JavaSript弱类型且动态类型 早前的JavaScript应用简单 JavaScript没有编译环节

# 强类型的优势

  • 错误更早暴露
  • 代码更加智能，编码更准确
  • 重构更牢靠
  • 减少不必要的类型判断

# 可选参数和默认参数

在TypeScript里我们可以在参数名旁使用 ?实现可选参数的功能 可选参数和默认参数必须跟在必须参数后面

function buildName(firstName: string, lastName?: string) {
    if (lastName)
        return firstName + " " + lastName;
    else
        return firstName;
}

默认参数

function buildName(firstName: string, lastName:string = "Smith") {
    // ...
}

函数表达式

const func: (a: number, b: number) => string = (a: number, b: number): string => {
    return 'func'
}

# 类型断言

类型断言好比其它语言里的类型转换，但是不进行特殊的数据检查和解构。 它没有运行时的影响，只是在编译阶段起作用。

const age = 19
const nums = [10, 2, 30, 102, 10]
const res = nums.find(item => item > 10)
const square = res * res//报错

// 使用类型断言
const num1 = res as number
const num2 = <number>res

# 类的访问修饰符

public (默认) 公共 private 私有，不能在声明它的类的外部访问 protected 与private修饰符的行为很相似，但有一点不同， protected成员在派生类中仍然可以访问 readonly 只读属性 构造函数也可以被标记成 protected。 这意味着这个类不能在包含它的类外被实例化，但是能被继承。

class Person {
    name: string
    private age: number
    protected readonly gender: boolean

    constructor(name: string, age: number) {
        this.name = name
        this.age = age
    }
    sayHi() {
        console.log(`Hi,I'm ${name}`)
    }
}
const person = new Person('tom', 10)
console.log(person.name)
console.log(person.age)//报错  属性“age”为私有属性，只能在类“Person”中访问
console.log(person.gender)//报错  属性“gender”受保护，只能在类“Person”及其子类中访问
class Student extends Person {
    constructor(name: string, age: number) {
        super(name, age)
        console.log(this.gender)//可以访问
    }
}

# 抽象类

抽象类做为其它派生类的基类使用。 它们一般不会直接被实例化。 不同于接口，抽象类可以包含成员的实现细节。 abstract关键字是用于定义抽象类和在抽象类内部定义抽象方法。

abstract class Animal {
    eat(food: string): void {
        console.log(`eat ${food}`)
    }
    abstract run(distance: number): void
}
class Dog extends Animal {
    run(distance: number): void {
        console.log(`run ${distance}`)
    }

}
const dog = new Dog()
dog.eat('apple')
dog.run(100)
