# 规范化标准

# 规范化介绍

为什么要有规范化标椎 软件开发需要多人协调 不同开发者具有不同的编码习惯和喜好 不同的喜好增加项目的维护成本 每个项目或团队需要明确统一的标准

哪里需要规范化标椎 代码、文档、甚至提交日志 开发过程中人为编写的成果物 * 代码规范化最为重要

实施规范化的方法 编码前人为的标准约定 通过工具实现lint

常见的规范化实现方式 ESLint工具使用 定制ESLint校验规则 ESLint对TypeScript的支持 ESLint结合自动化工具或者Webpack 基于ESLint的衍生工具 Stylelint工具的使用

# ESLint

# ESLint介绍

  • 最为流行的JavaScript Lint工具监测JS代码质量
  • ESLint很容易同意开发者的编码风格
  • ESLint可以帮助开发者提升编码能力 ESLint安装
npm install eslint --save-dev
eslint --init
npx eslint src/*.js

修复代码中的部分错误

npx eslint --fix

# 结合gulp的使用

结合gulp的使用(使用gulp-eslint)

const script = () => {
  return src('src/assets/scripts/*.js', {
      base: 'src'
    })
    .pipe(plugins.eslint())
    .pipe(plugins.eslint.format())
    .pipe(plugins.eslint.failAfterError())
    .pipe(plugins.babel({
      presets: ['@babel/preset-env']
    }))
    .pipe(dest('temp'))
    .pipe(bs.reload({
      stream: true
    }))
}

        test: /\.js$/,

      template: 'src/index.html'
    })
  ]

# Prettier的使用

npm install prettier --save-dev

格式化单个文件

npx prettier style.css

格式化文件并将格式化后的代码覆盖源文件

npx prettier style.css --write

格式化当前目录的所有文件

npx prettier . --write

不要太依赖prettire格式化我们的代码，我们编码过程中就应该写书格式良好的代码

# Git Hooks

通过githooks在代码提交之前强制执行lint

Git Hooks介绍 Git Hooks也被称为钩子，每个钩子都对应一个任务 通过shell脚本可以编写出钩子任务触发时要具体执行的操作

# ESLint结合Git Hooks

Husky可以实现Git Hooks的使用需求

npm install husky --save-dev

在package.json文件中配置husky

...
  "husky": {
    "hooks": {
      "pre-commit": "npm run precommit"//提交之前要做的操作
    }
  }
...

如果想要检查之后或检查之前，做些其他事情，这时候需要使用到lint-staged

在package.json文件中配置lint-staged

"scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "commit": "git-cz",
    "eslint": "eslint 1.js",
    "prettier": "prettier . --write",
    "precommit": "lint-staged"
  },
...
  "husky": {
    "hooks": {
      "pre-commit": "npm run precommit"
    }
  },
  "lint-staged": {
    "*.js": [
      "npm run eslint",
      "git add"
    ]
  }
