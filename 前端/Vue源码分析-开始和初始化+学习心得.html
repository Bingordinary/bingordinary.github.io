打包工具Rollup * vue源码使用的打包工具是Rollup,比webpack轻量，速度更快 * webpack会把所有文件当做模块，Rollup只处理js文件，所以更适合在类库中使用 * Rollup打包过程中不会生产冗余代码 首先进入vue源码目录下 1. 安装依赖

npm install
  1. 设置sourcemap,在package.json dev脚本中增加参数--sourcemap
"dev": "rollup -w -c scripts/config.js --sourcemap --environment TARGET:web-full-dev"
  1. 执行npm run dev执行打包，用rollup打包工具
  2. -w参数监听文件变化文件更新自动重新打包
  3. -c是设置配置文件
  4. --environment设置环境变量
npm run dev

vue的不同构建版本

  • 完整版：同时包含编译器和运行时的版本。

  • 编译器：用来将模板字符串编译成为 JavaScript 渲染函数的代码，体积大、效率低。

  • 运行时：用来创建 Vue 实例、渲染并处理虚拟 DOM 等的代码，体积小、效率高。基本上就是除去编译器的代码。 因此生产环境推荐使用运行时版本，因为运行时版本相比完整版体积要小大约 30%

  • UMD：UMD 版本通用的模块版本，支持多种模块方式。vue.js 默认文件就是运行时 + 编译器的UMD 版本

  • CommonJS(cjs)：CommonJS 版本用来配合老的打包工具比如 Browserify 或 webpack 1。

  • ES Module：从 2.6 开始 Vue 会提供两个 ES Modules (ESM) 构建文件，为现代打包工具提供的 版本。

  • ESM 格式被设计为可以被静态分析，所以打包工具可以利用这一点来进行“tree-shaking”并 *将用不到的代码排除出最终的包。

  • ES6 模块与 CommonJS 模块的差异 (opens new window)

基于 Vue-CLI 创建的项目默认使用的是 vue.runtime.esm.js

Vue.prototype.$mount = function (
  el?: string | Element,
  hydrating?: boolean
): Component {
  el = el && query(el)

  /* istanbul ignore if */
  // 如果el是document.body或 document.documentElement直接返回Vue实例
  if (el === document.body || el === document.documentElement) {
    process.env.NODE_ENV !== 'production' && warn(
      `Do not mount Vue to <html> or <body> - mount to normal elements instead.`
    )
    return this
  }

  const options = this.$options
  // resolve template/el and convert to render function
  // 如果没有render函数，
  // 就判断有没有template,如果没有就把el当成template,  把template转换成render函数
  // 
  if (!options.render) {
    let template = options.template
    if (template) {
      if (typeof template === 'string') {
        if (template.charAt(0) === '#') {
          template = idToTemplate(template)
          /* istanbul ignore if */
          if (process.env.NODE_ENV !== 'production' && !template) {
            warn(
              `Template element not found or is empty: ${options.template}`,
              this
            )
          }
        }
      } else if (template.nodeType) {
        template = template.innerHTML
      } else {
        if (process.env.NODE_ENV !== 'production') {
          warn('invalid template option:' + template, this)
        }
        return this
      }
    } else if (el) {
      template = getOuterHTML(el)
    }
    if (template) {
      /* istanbul ignore if */
      if (process.env.NODE_ENV !== 'production' && config.performance && mark) {
        mark('compile')
      }

      const { render, staticRenderFns } = compileToFunctions(template, {
        outputSourceRange: process.env.NODE_ENV !== 'production',
        shouldDecodeNewlines,
        shouldDecodeNewlinesForHref,
        delimiters: options.delimiters,
        comments: options.comments
      }, this)
      options.render = render
      options.staticRenderFns = staticRenderFns

      /* istanbul ignore if */
      if (process.env.NODE_ENV !== 'production' && config.performance && mark) {
        mark('compile end')
        measure(`vue ${this._name} compile`, 'compile', 'compile end')
      }
    }
  }
  return mount.call(this, el, hydrating)
}

// 获取el的outerHTML,如果有直接返回，如果没有，将el（所有节点）拷贝到新创建的一个div标签中，并返回新建建div标签的innerHTML
function getOuterHTML (el: Element): string {
  if (el.outerHTML) {
    return el.outerHTML
  } else {
    const container = document.createElement('div')
    container.appendChild(el.cloneNode(true))
    return container.innerHTML
  }
}

通过阅读源码，$mout执行主要流程 1. 判断el是否是document.body或 document.documentElement，没有直接返回vue实例(不走下面流程)，因此el不能是document.body或 document.documentElement 2. 判断有没有render函数 1. 没有，判断render函数判断有没有template 1. 没有template，就把template对应的html（getOuterHTML函数获取） 当成模板 2. 有，template就是template.innerHTML 1. 有，什么也不做 3. 执行mount挂载dom

export function initGlobalAPI (Vue: GlobalAPI) {
  // config
  const configDef = {}
  configDef.get = () => config
  if (process.env.NODE_ENV !== 'production') {
    configDef.set = () => {
      warn(
        'Do not replace the Vue.config object, set individual fields instead.'
      )
    }
  }
  Object.defineProperty(Vue, 'config', configDef)

  // exposed util methods.
  // NOTE: these are not considered part of the public API - avoid relying on
  // them unless you are aware of the risk.

    warn,
    extend,
    mergeOptions,
    defineReactive
  }

  Vue.set = set
  Vue.delete = del
  Vue.nextTick = nextTick

  // 2.6 explicit observable API
  Vue.observable = <T>(obj: T): T => {
    observe(obj)
    return obj
  }

  // 给Vue.options增加成员components/directive/filters,初始值为空对象
  Vue.options = Object.create(null)
  ASSET_TYPES.forEach(type => {
    Vue.options[type + 's'] = Object.create(null)

  // this is used to identify the "base" constructor to extend all plain-object
  // components with in Weex's multi-instance scenarios.
  Vue.options._base = Vue

  initMixin(Vue)

在src/core/global-api/use.js里定义了initUse方法

import { toArray } from '../util/index'

export function initUse (Vue: GlobalAPI) {
  Vue.use = function (plugin: Function | Object) {
    const installedPlugins = (this._installedPlugins || (this._installedPlugins = []))
    // 如果插件已经安装，直接返this
    if (installedPlugins.indexOf(plugin) > -1) {
      return this
    }

    // additional parameters
    // 把arguments的第一个参数去掉 ,剩下的就是安装插件需要传递的参数,例如 Vue.use(pluginName,a,b,c...) a,b,c...就是args
    const args = toArray(arguments, 1)
    // 将Vue插入到参数的第一个元素
    args.unshift(this)
    if (typeof plugin.install === 'function') {
      plugin.install.apply(plugin, args)
    } else if (typeof plugin === 'function') {
      plugin.apply(null, args)
    }
    // 保存已安装的插件
    installedPlugins.push(plugin)
    return this
  }
}

src/core/global-api/extend.js定义了initExtend，定义了Vue.extend方法。

Vue.extend方法的用法是使用基础 Vue 构造器，创建一个“子类”。参数是一个包含组件选项的对象。

import {
  ASSET_TYPES
} from 'shared/constants'
import {
  defineComputed,
  proxy
} from '../instance/state'
import {
  extend,
  mergeOptions,
  validateComponentName
} from '../util/index'

export function initExtend(Vue: GlobalAPI) {
  /**
   * Each instance constructor, including Vue, has a unique
   * cid. This enables us to create wrapped "child
   * constructors" for prototypal inheritance and cache them.
   */
  Vue.cid = 0
  let cid = 1

  /**
   * Class inheritance
   */
  // 返回组件的构造函数
  Vue.extend = function (extendOptions: Object): Function {
    extendOptions = extendOptions || {}
    const Super = this
    const SuperId = Super.cid
    const cachedCtors = extendOptions._Ctor || (extendOptions._Ctor = {})
    if (cachedCtors[SuperId]) {
      return cachedCtors[SuperId]
    }

    const name = extendOptions.name || Super.options.name
    if (process.env.NODE_ENV !== 'production' && name) {
      validateComponentName(name)
    }
    //定义组件的构造函数
    const Sub = function VueComponent(options) {
      this._init(options)
    }
    // 组件继承Vue的原型
    Sub.prototype = Object.create(Super.prototype)
    // 组件的constructor指向自己
    Sub.prototype.constructor = Sub
    Sub.cid = cid++
    Sub.options = mergeOptions(
      Super.options,
      extendOptions
    )
    Sub['super'] = Super

    // For props and computed properties, we define the proxy getters on
    // the Vue instances at extension time, on the extended prototype. This
    // avoids Object.defineProperty calls for each instance created.
    if (Sub.options.props) {
      initProps(Sub)
    }
    if (Sub.options.computed) {
      initComputed(Sub)
    }

    // allow further extension/mixin/plugin usage
    Sub.extend = Super.extend
    Sub.mixin = Super.mixin
    Sub.use = Super.use

    // create asset registers, so extended classes
    // can have their private assets too.
    ASSET_TYPES.forEach(function (type) {
      Sub[type] = Super[type]
    })
    // enable recursive self-lookup
    if (name) {
      Sub.options.components[name] = Sub
    }

    // keep a reference to the super options at extension time.
    // later at instantiation we can check if Super's options have
    // been updated.
    Sub.superOptions = Super.options
    Sub.extendOptions = extendOptions
    Sub.sealedOptions = extend({}, Sub.options)

    // cache constructor
    cachedCtors[SuperId] = Sub
    return Sub
  }
}

src/core/global-api/extend.js定义了initAssetRegisters方法

initAssetRegisters方法定义了Vue的 directive component filter静态方法

代码执行主要过程分析 如果definition没有传递,直接返回this.options[type + 's'][id]

[Vue.directive( id, [definition] )] ，definition类型为{Function | Object}

  • 如果definition是原始对象，就调用Vue.extend 把组件对象转换为组件的构造函数
  • 为Function 就返回definition

Vue.filter( id, [definition] )， definition类型为{Function}

  • 直接返回definition

Vue.component( id, [definition] )， {Function | Object}

  • 如果definition是函数，将设置为definition { bind: definition, update: definition } * 为Functiion 直接返回definition
src/core/global-api/extend.js
import { ASSET_TYPES } from 'shared/constants'
import { isPlainObject, validateComponentName } from '../util/index'

export function initAssetRegisters (Vue: GlobalAPI) {
  /**
   * Create asset registration methods.
   */
  // ASSET_TYPES  directive component filter
  ASSET_TYPES.forEach(type => {
    Vue[type] = function (
      id: string,
      definition: Function | Object
    ): Function | Object | void {
      if (!definition) {
        return this.options[type + 's'][id]
      } else {
        /* istanbul ignore if */
        if (process.env.NODE_ENV !== 'production' && type === 'component') {
          validateComponentName(id)
        }
        // 是组件 并且 definition是原始对象
        if (type === 'component' && isPlainObject(definition)) {
          definition.name = definition.name || id
          //调用Vue.extend 把组件对象转换为组件的构造函数
          definition = this.options._base.extend(definition)
        }
        // 指令  如果definition是函数，将函数设置为对象{ bind: definition, update: definition }
        if (type === 'directive' && typeof definition === 'function') {
          definition = { bind: definition, update: definition }
        }
        this.options[type + 's'][id] = definition
        return definition
      }
    }
  })
}
